### 1. Configure MineVerify for your app

```yaml
apps:
  my-app:
    name: "My App"
    base-url: "https://my-app.com"
    token: "generated-token"
    poll-interval-seconds: 3
```

### 2. Store the same token in your app backend

```env
MINEVERIFY_TOKEN=generated-token
```

### 3. Reject requests without this header

```http
Authorization: Bearer generated-token
```

### 4. Store verification requests with these fields

| Field | Required | Notes |
| --- | --- | --- |
| `requestId` | Yes | Unique id generated by your app. |
| `externalUserId` | Yes | Your logged-in app user id. |
| `state` | Yes | `pending_code`, `pending_validation`, `validated`, or `expired`. |
| `code` | No | Filled by `/api/mineverify/code-created`. |
| `expiresAt` | No | Filled by `/api/mineverify/code-created`. |
| `minecraftUuid` | No | Filled by `/api/mineverify/validated`. |
| `minecraftName` | No | Filled by `/api/mineverify/validated`. |

### 5. Create a request when the user starts verification

```json
{
  "requestId": "018f4f58-6fb7-7f65-bd2a-8a6f7c83f8e1",
  "externalUserId": "user_123",
  "state": "pending_code",
  "code": null,
  "expiresAt": null,
  "minecraftUuid": null,
  "minecraftName": null
}
```

### 6. Implement `GET /api/mineverify/pending-requests`

Return only non-expired requests in `pending_code`.

```json
{
  "requests": [
    {
      "requestId": "018f4f58-6fb7-7f65-bd2a-8a6f7c83f8e1"
    }
  ]
}
```

Return an empty list when no request is waiting.

```json
{
  "requests": []
}
```

### 7. Implement `POST /api/mineverify/code-created`

Expected payload:

```json
{
  "appId": "my-app",
  "requestId": "018f4f58-6fb7-7f65-bd2a-8a6f7c83f8e1",
  "code": "K7M9-P2Q4",
  "expiresAt": "2026-06-04T16:05:00Z"
}
```

Required handling:

```text
find request by requestId
require state pending_code
store code
store expiresAt
set state pending_validation
return 2xx
```

### 8. Show the command to the user while the request is `pending_validation`

```text
/mineverify K7M9-P2Q4
```

### 9. Implement `POST /api/mineverify/validated`

Expected payload:

```json
{
  "appId": "my-app",
  "requestId": "018f4f58-6fb7-7f65-bd2a-8a6f7c83f8e1",
  "code": "K7M9-P2Q4",
  "minecraftUuid": "6f8f5771-8ec8-4b8d-bc40-8cbe2f84f5a3",
  "minecraftName": "PlayerName",
  "validatedAt": "2026-06-04T16:02:20Z"
}
```

Required handling:

```text
find request by requestId
require state pending_validation
require matching code
require expiresAt not passed
store minecraftUuid
store minecraftName
set state validated
persist externalUserId <-> minecraftUuid link
return 2xx
```

### 10. Expire stale requests in your app

```text
pending_code -> expired
pending_validation -> expired
```

### 11. Browser UI should only use your app backend states

| State | UI |
| --- | --- |
| `pending_code` | Waiting for Minecraft server. |
| `pending_validation` | Show `/mineverify <code>`. |
| `validated` | Show linked Minecraft account. |
| `expired` | Ask the user to start again. |
